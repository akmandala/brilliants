Snippet Database

A Snippet Database for turning sketches into buildable hardware.

Hardware Snippets are reusable design blocks stored with a real BoM, clear wiring intent, and library-ready part mappings. The LLM reads your sketch/spec, infers requirements, and selects the best-fit Snippets from the database — it does not generate circuits on the fly.

What a Hardware Snippet contains

  • Real BoM: MPNs, key params, alternates (when available), lifecycle/supply notes
  • Wiring intent: how the block connects in the schematic (topology + boundaries)
  • Constraints + calculations: assumptions, margins, derating where it matters
  • Simulation hooks: ready-to-run starting points when applicable
  • Part library mapping: symbols/footprints/3D links or delivered Altium libraries

That structure is what makes Snippets reusable — and what makes selection reliable.

Compute selection is first-class

Selecting a sensor or a regulator is often straightforward. Selecting the right MCU/processor is not. Our database stores compute Snippets with the surrounding system blocks already defined, so selection includes what actually makes the design work.

  • MCU Snippets: power + clocks + debug + interfaces
  • Processor/SoM Snippets: PMIC/rails + DDR + boot storage + high-speed I/O
  • Camera/vision paths: interface patterns + constraints (where relevant)
  • Companion blocks: ESD/EMI, connectors, filtering, safety/protection patterns

Status: database + assisted selection available now. Productised UI/API in progress.

Why database-first wins

  • Faster starts: reuse validated blocks instead of rewriting the same circuits
  • Fewer respins: assumptions and constraints are explicit
  • Better handoff: BoM + library mapping reduces assembly ambiguity
  • Better decisions: compute choices are justified, not guessed
